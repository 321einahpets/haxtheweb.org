HAX is short for Headless Authoring Experience

It is a next generation WYSIWYG that's built in Web components and can talk to any web component / HTML tag in it's body area so long as it emits HAXschema; a simple JSON object.

Why HAX?

There's no reason to tightly couple editing experience to the platform you are working on. Justify to a customer why a video-player on a web page requires a lot of work on the backend in order for them to leverage it. It's a hard sell even if it's justified.
