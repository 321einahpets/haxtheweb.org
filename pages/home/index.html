HAX is the only editor that frees you of editor specific code when doing design work. This is possible because to extend HAX visually (see Make button pinwheel at top) you simply do the following:

  1. Make a new web component in any library you want
  2. Add some simple, documented HAXSchema
  3. When  HAX is around, it'll know how to edit your element but you can use it anywhere else regardless!

HAX is easy for developers to integrate to. Yarn / NPM install the element, place the editable content inside the slot of the h-a-x tag and now you've got an incredible, flexible and lack of lock-in based editing experience. Go forth and HAX the web!

As an example, that video-player at the top of the page was completely designed and worked on without ever thinking about HAX. Then at the end, when the API is done, we wrote some small HAXSchema and now it's editable and able to be placed in HAX.
